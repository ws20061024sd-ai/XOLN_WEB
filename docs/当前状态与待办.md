## 一、作品栏目分类重组（方案 B，准备实现）

**选定方案**：catch-all 路由 `works/[...path]/page.tsx`，支持任意深度嵌套。

- `/works` → 列出所有顶级系列和独立作品
- `/works/设计` → 系列子页面，显示该系列下的作品
- `/works/设计/logo` → 任意深度的文章详情
- 内容层用子目录 = 文件夹，`content/works/设计/xxx.md`
- 内容编辑方式不变：右键新建文件夹 + `.md` 文件

**实现涉及**：
- `content.ts` 新增递归读取 + 路径定位函数
- `src/app/works/page.tsx` 改为系列目录视图
- 新增 `src/app/works/[...path]/page.tsx` catch-all 路由
- 旧的 `[section]/[slug]` 路由会自动被 works 的专用路由覆盖
