## 一、待讨论：共创栏目

### 需求
- 新栏目"共创"，读者投稿写文章
- 独立导航入口，不与其他栏目混排
- 列表+详情页面，像观念栏一样
- 读者可用 Markdown 写作（支持图片/视频/Bilibili/YouTube）
- 提交后需你审核通过才公开显示

### 架构约束

项目是纯静态导出（`output: 'export'`），所有现有栏目的页面在构建时从本地 `.md` 文件生成。但共创文章存在 API 数据库里，构建时不知道有哪些 ID，所以**不能用和观念栏完全相同的路由方案**。

### 前端方案

**路由设计**：
- `/community` — 单页面，客户端组件，同时承载列表和详情两种视图
- 列表模式：展示所有已审核文章（标题/作者/日期/标签）
- 点击某篇文章 → 同一页面切换到详情模式，渲染正文
- URL 用 `?id=xxx` 区分详情页（`/community?id=15`，静态度兼容）

**写作表单**：
- 在列表页顶部或独立区域有个"我要投稿"按钮
- 展开表单：标题、作者名、正文（Markdown 文本域）、可选标签
- 支持实时预览（复用 MarkdownRenderer）
- 提交后显示"投稿已提交，审核后显示"

**渲染**：正文直接用现有 MarkdownRenderer，图片/视频/Bilibili/YouTube 全部无缝支持。

### 后端方案

**数据库表** `community_posts`：
```
id         INTEGER PRIMARY KEY AUTOINCREMENT
title      TEXT NOT NULL
author     TEXT NOT NULL
content    TEXT NOT NULL          -- Markdown 原文
tags       TEXT                   -- 逗号分隔，可选
approved   INTEGER DEFAULT 0     -- 0=待审核, 1=已通过
created_at DATETIME DEFAULT CURRENT_TIMESTAMP
```

**API 接口**：
```
GET   /api/community         公开：获取已审核文章列表
GET   /api/community/:id     公开：获取单篇文章详情
POST  /api/community         公开：提交投稿（限流）
PATCH /api/admin/community/:id  管理：审核通过
DELETE /api/admin/community/:id 管理：删除
```

**反垃圾**：沿用现有限流（每 IP 每秒 5 次）+ 内容长度校验（标题 ≤100 字，正文 ≤50000 字）+ 默认待审核。

### 管理面板

AdminPanel 新增"共创审核" Tab，列出待审核文章，支持预览正文、通过、删除。

### 导航

Header 导航栏新增"共创"入口，放在"留言板"旁边或后面，独立于现有栏目。

### 待定问题

1. **投稿是否需要登录**？目前设计是匿名投稿（填作者名即可），和留言板一致。如果需要 GitHub OAuth 登录，复杂度会大幅增加。
2. **正文编辑器**：纯 Markdown 文本域还是需要简单的所见即所得？Markdown 文本域最简单，但对不熟悉 Markdown 的读者不友好。
3. **共创首页要不要出现在网站首页卡片区**？类似留言板那样有一个引导卡片。

## 四、本地开发注意事项

- 启动命令：`npm run dev -- --port 3099`（不是 `npm run dev`，默认是 3000）
- 打不开时按以下顺序排查：
  1. `lsof -i :3099` 看进程是否在
  2. `curl http://localhost:3099` 看是否响应
  3. 连接但不响应 → `kill -9 PID && rm -rf .next && npm run dev -- --port 3099`
  4. 端口没进程 → 直接启动
- 开发审查清单：[docs/开发审查清单.md](docs/开发审查清单.md)
