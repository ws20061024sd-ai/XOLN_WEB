# 项目复盘：从零到上线的完整路径

> 2026-05-20。记录整个博客网站从构思到上线的全过程、遇到的问题和最终的解决方式。

---

## 一、项目概述

用 Next.js 16 + Tailwind CSS v4 构建纯静态个人网站，前端托管在腾讯云 COS + CDN，后端运行在腾讯云 Lighthouse 上提供评论、统计等动态功能。

建站结束状态：
- 前端 33 个静态页面，7 个栏目
- 后端 12 个 API 接口
- 管理面板可操作评论、消息、统计
- `https://xolnxoln.cn` 正常运行

## 三、遇到的问题和解决方法

### 问题 1：ScrollReveal 导致页面内容不可见

**现象**：页面标题可见，往下的内容全部空白。

**原因**：`ScrollReveal` 组件默认 `useState(false)`，内容初始 `opacity: 0`。当包裹的 MarkdownRenderer 内容触发 React hydration 异常时，React 卸载重挂组件，IntersectionObserver 被销毁，内容永远卡在透明。

**解决**：改为 `useState(true)`——内容默认可见。JS 运行时检查元素位置：在视口内直接显示，在视口外才先隐藏后等滚动触发。

**教训**：动画组件不应该用隐藏作为初始状态。**内容默认可见，动画只做增强。**

### 问题 2：better-sqlite3 跨平台不兼容

**现象**：Docker 容器启动报错 `Exec format error`。

**原因**：`better-sqlite3` 是 C 原生模块，Mac（ARM64）上编译的二进制无法在 Linux（x86_64）上运行。Docker 内重新编译需要 `gcc`、`make`、`python3`，下载编译工具就要 10 分钟。

**解决**：换用 `sql.js`——纯 JavaScript 的 SQLite，不需要编译，不挑 CPU 架构。Docker 构建从 10 分钟降到 10 秒。个人博客的访问量完全感知不到性能差异。

**教训**：**能用纯 JS 就别用原生编译的库。** 跨平台部署时原生模块是最大的坑。

### 问题 3：Docker 缓存不更新代码

**现象**：改了代码后服务器上没变化。

**原因**：`docker compose restart` 只重启旧容器，不会重新构建镜像。必须 `docker compose up -d --build`。

### 问题 4：CORS 跨域

**现象**：浏览器报 `Failed to fetch`。

**原因**：后端 CORS 白名单没包含 `localhost:3099`（开发环境）。

**解决**：Hono 的 `cors` 中间件添加开发环境的 origin。

### 问题 5：HTTPS 环境变量不匹配

**现象**：`.env.local` 写了 `NEXT_PUBLIC_API_URL=https://api.xolnxoln.cn`，浏览器请求失败。

**原因**：服务器只有 HTTP（80 端口），没配 SSL，浏览器发 `fetch` 去 443 端口什么都没有。开发阶段改为 `http://` 即可。

### 问题 6：评论不显示（approved 默认值）

**现象**：评论提交成功但页面看不到。

**原因**：数据库表在建表时 `approved DEFAULT 0`。INSERT 语句没有显式设 `approved` 值，SQLite 使用旧默认值 0。查询用 `WHERE approved = 1`，评论被过滤。

**解决**：INSERT 时显式写 `approved = 1`。

**教训**：**`CREATE TABLE IF NOT EXISTS` 不会修改已有表的默认值。** 建表语句改了但表已存在时不会生效。

### 问题 7：COS 文件上传路径错误

**现象**：COS 桶里文件在 `out/` 子目录下，根路径访问不到 `index.html`。

**原因**：上传脚本中 `outDir = './out'`，文件路径 `out/index.html`，用 `file.replace('./out/', '')` 替换时 `./out/` 和 `out/` 不匹配。

**解决**：改为 `file.replace('out/', '')`。同时先清掉桶里的 `out/` 前缀旧文件。

**教训**：路径处理要注意 `./` 前缀。

### 问题 8：COS 下载文件而不是显示网页

**现象**：访问网站直接下载 HTML 文件。

**原因**：上传时没有设 `Content-Type`，COS 默认把 HTML 当二进制文件，浏览器触发下载。

**解决**：上传时根据文件扩展名设置对应的 MIME 类型（`.html` → `text/html`，`.css` → `text/css` 等）。

### 问题 9：CDN 回源 403——最折腾的问题

**现象**：CDN 配置完成后访问域名返回 403 AccessDenied，直接访问 COS 网站端点正常。

**排查过程**：
1. 以为是 DNS 没生效 → DNS 解析正确
2. 以为是 COS 桶权限 → 直接访问桶是 200
3. 以为是回源 HOST 没配 → COS 源类型回源 HOST 自动填，文档说不可修改
4. 以为是 CDN 源站类型选错 → 是 COS 源，地址正确

**真正原因**：COS 有两种访问端点：
- 标准端点 `cos.ap-guangzhou.myqcloud.com`：访问 `/` 返回 403，不支持首页索引
- 网站端点 `cos-website.ap-guangzhou.myqcloud.com`：访问 `/` 自动返回 `index.html`

CDN 配的是标准端点。当用户访问 `xolnxoln.cn/` 时，CDN 去 COS 标准端点请求 `/`，COS 返回 403。

**解决**：CDN 源站从"默认域名"切换为"静态网站"（对应网站端点）。

**教训**：**COS 标准端点是 API 端点，网站托管必须用网站端点。** 区分清楚两个端点的作用。

### 问题 10：备案期间域名被拦截

**现象**：`api.xolnxoln.cn` 被腾讯云 DNS 拦截，跳转到备案提示页。

**原因**：ICP 备案提交后到审批通过前，域名被临时拦截。

**解决**：备案期间 Nginx 关机（`systemctl stop nginx`），通过后恢复。管理面板临时用 IP 直连。

### 问题 11：Nginx SSL 配置被 certbot 覆盖后又切 CDN

**现象**：certbot 给 Lighthouse 上的 Nginx 配了 SSL，后来 DNS 切到 CDN 后 CDN 自带证书。

**结果**：两边都有 SSL，CDN 的生效，Lighthouse 上的证书继续自动续期（不影响，但也没用上）。

## 四、走弯路的教训

1. **DNS 反复改**：在 COS 直连和 CDN 之间反复调整 DNS 记录。正确的路一开始就是 CDN → COS 网站端点。
2. **没先看文档就动手**：让用户切"自有源"来填回源 HOST，实际上 COS 源的静态网站选项就能解决。
3. **没有完整诊断就下结论**：多次以为是不同原因，最终才定位到"标准端点 vs 网站端点"的区别。
4. **上传脚本路径问题**：`./out` 和 `out` 路径不一致导到文件跑到了子目录。
5. **Content-Type 反复踩坑**：三次上传才设对文件类型。

##五、最终架构

```
用户 → xolnxoln.cn
       ↓ DNS CNAME
       CDN（HTTPS 证书由 CDN 提供）
       ↓ 回源 HTTP
       COS 网站端点（自动首页索引）
       ↓
       静态 HTML/CSS/JS

用户 → api.xolnxoln.cn
       ↓ DNS A 记录
       Lighthouse Nginx
       ↓ 反代
       Docker 容器（Hono + sql.js）
```

## 六、有用的技术决策

| 决策 | 为什么 |
|------|--------|
| 纯 Markdown | 不用 CMS，.md 文件直接用 IDE 写，git 版本管理 |
| 列表+详情架构 | 每栏目拆成列表页和详情页，不堆在一页 |
| order 排序 | frontmatter 里数字控制文章顺序 |
| sql.js 而不是 better-sqlite3 | 纯 JS，不挑 CPU 架构，Docker 部署省心 |
| CDN + COS 而不是 Lighthouse 反代 | 反代多一层复杂度，CDN 自带 SSL 且全球加速 |
| API_BASE 环境变量 | 前后端解耦，后端不存在时前端正常展示 |
