## 二、SSL 证书

你的网站有两张独立的 SSL 证书，互不影响。一张过期不影响另一张。

### 证书一：CDN 前端证书（TrustAsia，90 天免费）

**覆盖域名**：`xolnxoln.cn`、`www.xolnxoln.cn`

**当前到期**：2026-08-09（腾讯云会提前短信/邮件/站内信提醒）

**在哪管理**：[腾讯云 SSL 证书控制台](https://console.cloud.tencent.com/ssl)

**怎么续期**（到期前 30 天内操作）：

1. 打开 [SSL 证书控制台](https://console.cloud.tencent.com/ssl)，找到 `xolnxoln.cn` 的证书，状态显示"即将过期"
2. 点击**续期**按钮
3. 选择验证方式 → **DNS 验证**（推荐，最快）：
   - 腾讯云会自动生成一条 DNS TXT 记录
   - 点击"一键添加"，系统自动加到 DNSPod
   - 等 1-5 分钟，点"验证"
4. 验证通过后，证书自动签发
5. 签发后，CDN 通常会自动关联新证书。如果没有，去 [CDN 控制台](https://console.cloud.tencent.com/cdn) → 域名管理 → `xolnxoln.cn` → HTTPS 配置 → 选择新证书 → 保存
6. 全程 5-10 分钟，不需要改任何代码

**过期会怎样**：
- 用户浏览器打开 `https://xolnxoln.cn` 会显示"不安全"警告
- 网站内容不受影响，HTTP 仍然可用（但 CDN 已强制 HTTPS 跳转，所以用户基本打不开）
- 新证书签发后立即恢复

**注意**：这个证书不是自动续期的。TrustAsia 免费证书每次都要手动操作。

---

### 证书二：API 后端证书（Let's Encrypt，90 天免费）

**覆盖域名**：`api.xolnxoln.cn`

**当前到期**：2026-08-21（certbot 应在此之前自动续期）

**在哪运行**：Lighthouse 服务器上的 certbot

**自动续期机制**：certbot 安装时配置了 systemd 定时任务，每天检查两次，到期前 30 天自动续。

**检查自动续期是否正常**（WebShell）：
```bash
systemctl status certbot-renew.timer
# 显示 active (running) = 正常
# 显示 inactive 或 failed = 需要修复
```

**手动续期**（如果自动续期失败）：
```bash
certbot renew --nginx
systemctl reload nginx
```

**查看到期时间**（WebShell）：
```bash
certbot certificates
```

**什么情况会导致自动续期失败**：
- 服务器关机超过 90 天（timer 从未触发）
- Nginx 配置被改，`/.well-known/acme-challenge/` 路径失效
- 域名 DNS 解析变了（certbot 需验证域名归属权）
- 服务器 80 端口被防火墙挡住（Let's Encrypt 验证需要 HTTP 80 端口）

**过期会怎样**：
- 前端调用 API 全部失败（评论/留言板/共创/统计/联系表单失效）
- 但静态页面不受影响，网站仍能访问
- 续期后无需重启，nginx reload 即可

**预防**：每月通过 WebShell 检查一次 `certbot certificates`。

---

### 快速检测命令（本地运行）

```bash
# 前端 CDN 证书
echo | openssl s_client -servername xolnxoln.cn -connect xolnxoln.cn:443 2>/dev/null | openssl x509 -noout -dates

# 后端 API 证书
echo | openssl s_client -servername api.xolnxoln.cn -connect api.xolnxoln.cn:443 2>/dev/null | openssl x509 -noout -dates
```

## 三、域名

### 域名在哪

腾讯云 → 域名管理 → `xolnxoln.cn`。

### 域名过期会怎样

- 到期当天：网站和 API 全部无法访问
- 到期 30 天内：可以续费恢复（可能有赎回费）
- 到期 30 天后：域名被释放，别人可以注册

### 怎么续费

腾讯云会提前发短信/邮件提醒。收到提醒后 → 域名管理 → 续费 → 选 1 年。

**建议**：一次性买 3-5 年，省得每年记。

## 四、Lighthouse 服务器

### 续费

Lighthouse 按月付费，每月自动从腾讯云账户扣款。确保账户余额充足（建议每月初检查一次）。

### 服务器过期会怎样

- API 停止工作
- 评论、留言板、联系表单全部失效
- 数据（SQLite 数据库）还在，续费后恢复
- **但前端网站不受影响**——静态页面在 COS 上，不依赖服务器

### 服务器续费前备份

SSH 进服务器：
```bash
cp /srv/blog-api/data/blog.db ~/blog-backup-$(date +%Y%m%d).db
```

## 五、数据库备份

### 什么时候备份

- 每次大改之前
- 每月一次定期备份
- 服务器到期前

### 怎么备份

SSH 进服务器：
```bash
# 查看数据库大小
ls -lh /srv/blog-api/data/blog.db

# 备份到 home 目录
cp /srv/blog-api/data/blog.db ~/blog-backup-$(date +%Y%m%d).db

# 下载到本地（Mac 终端，需要 SSH 通）
scp root@193.112.220.113:/srv/blog-api/data/blog.db ~/Desktop/blog-backup.db
```

### 数据库损坏怎么办

SQLite 单文件，备份就是复制文件。损坏后：
1. 停 Docker：`docker compose -f /srv/blog-api/docker-compose.yml down`
2. 替换 `blog.db` 为备份文件
3. 重启 Docker：`docker compose -f /srv/blog-api/docker-compose.yml up -d`

## 六、COS 和 CDN

### COS

- **费用**：按存储量 + 流量计费。正常博客访问每月不到 1 元。
- **文件被删**：重新 `npm run build && node upload.js` 上传。
- **桶权限被改**：确保是"公有读私有写"。

### CDN

- **状态检查**：CDN 控制台 → 域名管理 → 看两条域名是否"已部署"。
- **缓存刷新**：更新网站文件后必须刷新 CDN 缓存（控制台 → 缓存刷新 → 全部刷新）。
- **费用**：按 HTTPS 请求次数 + 流量计费。个人博客正常每月几毛钱到几块钱。
- **证书**：CDN 自带免费证书，腾讯云自动续期，不需要自己管。

## 九、常见紧急情况处理

### 网站打不开了

**第一步**：确认是哪种打不开
- 浏览器转圈 → DNS 或服务器问题
- 403 → CDN 或 COS 权限问题
- 404 → 文件不存在
- 502/503 → 后端 API 问题

**第二步**：按顺序排查
```bash
# 1. DNS 解析
nslookup xolnxoln.cn

# 2. COS 直连
curl "http://xolnxoln-1431302682.cos-website.ap-guangzhou.myqcloud.com/"

# 3. CDN
curl https://xolnxoln.cn

# 4. API
curl https://api.xolnxoln.cn/api/health
```

**第三步**：谁的问题修谁。
- DNS 不对 → 去 DNSPod 检查记录
- COS 不通 → 检查桶权限和文件
- CDN 不通 → 检查 CDN 状态和源站配置
- API 不通 → SSH 进服务器看 Docker 和 Nginx

### 被攻击/被刷

- 评论被刷：管理面板 → 评论 Tab → 删除
- API 被刷：服务器上有简易限流（每 IP 每秒 5 次），但不够。严重的话在 CDN 控制台开 WAF
- 数据库被灌满：SQLite 能存几百万条，个人博客不用担心

### 服务器磁盘满了

```bash
# SSH 进服务器
df -h
# 清理 Docker 无用镜像
docker system prune -a
```

## 十、Claude Code 对话上下文管理

### 问题

每轮对话有上下文限制（约 20 万字）。当对话积累到一定程度后，旧内容会被截断，Claude 会忘记早期的讨论。

### 征兆

- Claude 开始"忘记"之前做过的配置
- 你提到之前聊过的功能，Claude 要重新问
- 回复变得简短或和之前不一致

### 应对方式

**方式一：开新对话 + 加载项目记忆**

Claude Code 会自动读取项目根目录的 `CLAUDE.md` 文件作为项目指引。我已经建好了一份（如果还没有就创建一个），包含项目概览、常用命令、关键配置。新对话开始时 Claude 会自动读它，快速上手。

同时 `docs/` 下的 15 份文档包含了项目的全部上下文——任何新对话中，Claude 可以随时读这些文档获取信息。

**方式二：明确每次对话的目标**

开新对话时，直接告诉 Claude：
- "我是 XX，这个项目是我的个人博客"
- "项目目录在 `/Users/xoln/Desktop/编程/博客/site`"
- "我现在要做 XXX"

**方式三：利用 Git 提交信息**

每次改动都有 commit message，新对话里 `git log --oneline` 能快速回顾做了什么。

### 建议

- 当前这个对话已经很长了。下次有新任务可以开一个新对话。
- 这份《维护与注意事项》和《开发审查清单》本身就是为了让你和 Claude 在新对话中能快速对齐。
- `docs/当前状态与待办.md` 始终保持最新——新对话中 Claude 读它就知道全貌。
