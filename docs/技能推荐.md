# 技能推荐

> 实际使用标注：★ 表示已在项目中采用。

## Web Design

### 设计工具

| 工具 | 用途 | 推荐理由 |
|------|------|---------|
| [Figma](https://figma.com) | UI 设计/原型 | 免费额度够用，浏览器即开即用；社区有海量博客模板可参考 |
| [Excalidraw](https://excalidraw.com) | 手绘风示意图 | 博客文章里画架构图/流程图，手绘风比 Visio 亲切；开源、可自部署 |
| [v0.dev](https://v0.dev) | AI 生成 UI | Vercel 出品，文字描述生成 React/Tailwind 组件，快速起手稿 |
| [uizard](https://uizard.io) | 截图转设计 | 贴一张参考网站的截图，自动生成可编辑的设计稿 |

### 配色

| 工具 | 用途 |
|------|------|
| [Coolors](https://coolors.co) | 配色方案生成，按空格随机，锁定喜欢的继续调 |
| [Tailwind CSS Colors](https://tailwindcss.com/docs/customizing-colors) | Tailwind 内置 22 套色板（slate/gray/zinc/...），直接复用 |
| [Realtime Colors](https://realtimecolors.com) | 实时预览配色在 UI 组件上的效果 |
| [huemint](https://huemint.com) | 机器学习生成的配色方案，比人工调的质量高很多 |
| [OKLCH Color Picker](https://oklch.com) | 现代 CSS 色彩空间，比 hex/rgb 感知更均匀，Tailwind v4 原生支持 |

### 字体

| 工具 | 用途 |
|------|------|
| [Google Fonts](https://fonts.google.com) | 免费字体库，Noto Sans SC + Inter + JetBrains Mono 都在这 |
| [Fontsource](https://fontsource.org) | Google Fonts 的 npm 包版本，next/font 默认来源 |
| [Modern Font Stacks](https://modernfontstacks.com) | 按风格分类的系统字体栈（无衬线/衬线/等宽），不加载外部字体也能好看 |
| [uncut.wtf](https://uncut.wtf) | 精选小众免费字体，偏编辑/排版风格 |

### 图标

| 工具 | 用途 |
|------|------|
| [Lucide Icons](https://lucide.dev) | React 首选图标库，组件化调用，tree-shaking 按需加载 |
| [Heroicons](https://heroicons.com) | Tailwind 团队出品，风格克制，两个变体（outline/solid） |
| [Simple Icons](https://simpleicons.org) | 品牌/技术栈图标（GitHub/React/Next.js...），展示 "关于我" 页用 |
| [Phosphor Icons](https://phosphoricons.com) | 风格最多（6种变体），适合需要视觉差异化的场景 |

### 动效

| 工具 | 用途 |
|------|------|
| [tailwindcss-animate](https://github.com/jamiebuilds/tailwindcss-animate) | Tailwind 动画预设插件，一行 class 完事 |
| [motion](https://motion.dev)（原 Framer Motion） | React 动效库，适合复杂交互；个人博客可能用不上，但要动画就它 |
| [AutoAnimate](https://auto-animate.formkit.com) | 零配置，加一个 directive 自动给列表增删加过渡动画 |

## GitHub

### 静态托管与 CI/CD

| 工具 | 用途 | 推荐场景 |
|------|------|---------|
| [GitHub Actions](https://github.com/features/actions) | 自动化 CI/CD | push → build → 上传 COS，全套免费（公开仓库不限时长） |
| [GitHub Pages](https://pages.github.com) | 静态网站托管 | 备选部署方案，`username.github.io` 域名，和 GitHub Actions 深度集成 |
| [Vercel for GitHub](https://vercel.com/github) | 自动部署预览 | 每次 PR 自动生成预览 URL，适合团队协作；个人博客非必需 |
| [GitHub Webhooks](https://docs.github.com/en/webhooks) | 事件触发 | Phase 2 可实现：push 新文章 → webhook → Lighthouse 自动拉取构建 |

### 评论系统（基于 GitHub）

| 工具 | 原理 | 优劣 |
|------|------|------|
| [Giscus](https://giscus.app) | 用 GitHub Discussions 存评论 | 免费、无广告、数据归你自己仓库，Phase 1 最佳过渡方案 |
| [Utterances](https://utteranc.es) | 用 GitHub Issues 存评论 | 比 Giscus 更轻，但每个文章都要开一个 Issue |
| [gitalk](https://github.com/gitalk/gitalk) | 也是 Issues 存评论 | 需要 GitHub OAuth App，配置比 Giscus 复杂 |

Phase 1 用 Giscus，零成本把评论跑起来。Phase 2 自建后关掉。

### 写作与内容管理

| 工具 | 用途 |
|------|------|
| [Decap CMS](https://decapcms.org)（原 Netlify CMS） | 基于 Git 的 CMS，提供管理后台 UI，存到仓库里，保持纯静态 |
| [TinaCMS](https://tina.io) | 开源 Git-backed CMS，React 原生集成，比 Decap CMS 更现代但更重 |
| [Forestry](https://forestry.io) | 类似 Decap CMS，已停更但仍可用 |

对于个人博客，不装 CMS——直接 IDE 里写 .mdx → git push 就够了。

### 分析与监控

| 工具 | 用途 |
|------|------|
| [GitHub Traffic Insights](https://docs.github.com/en/repositories/viewing-traffic-to-a-repository) | 仓库自带流量分析（clones/views），仅限仓库层面 |
| [Umami](https://umami.is) | 开源、隐私友好的网站统计，可自部署到 Lighthouse |
| [Plausible](https://plausible.io) | 类似 Umami，更成熟的商业产品，有付费云版 |
| [GoatCounter](https://goatcounter.com) | 极简统计，一个二进制文件，部署极简 |

### GitHub API 与集成

| 工具 | 用途 |
|------|------|
| [gh CLI](https://cli.github.com) | 官方命令行工具，管理 Issues/PRs/Release，`gh api` 直接调 REST/GraphQL |
| [octokit](https://github.com/octokit) | GitHub 官方 JS SDK，适合在 Actions 或 API 里调用 GitHub 接口 |
| [GitHub GraphQL API](https://docs.github.com/en/graphql) | 比 REST 更灵活，一次请求精确拿所需数据 |
| [Shields.io](https://shields.io) | README 徽章生成，也适合博客里展示技术栈标签 |

### 开源许可

| 工具 | 用途 |
|------|------|
| [choosealicense.com](https://choosealicense.com) | GitHub 官方出品的许可证选择指南 |
| [GitHub License API](https://docs.github.com/en/rest/licenses) | 获取许可证模板文本 |

## 推荐组合（个人博客最优解）

```
写作      GitHub 仓库（.mdx 文件）+ VS Code
设计      不用 Figma，直接用 Tailwind 在代码里调
          → 配色用 huemint 选一套
          → 字体 Google Fonts 选两套
          → 图标 Lucide（React 组件化）
评论      Phase 1: Giscus → Phase 2: 自建
构建      GitHub Actions
部署      腾讯云 COS（静态前端）+ Lighthouse（API）
统计      Umami（自部署到 Lighthouse）
```
