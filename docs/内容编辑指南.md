# 内容编辑指南

> 本文档随网站变化实时更新。每次网站结构或编辑方式有变动，同步更新此文档。

## 先理解：这个网站是怎么工作的

你的网站由两个东西组成：

1. **内容**：你写的文字、放的图片。存在 `.md` 文件里，就像 Word 文档
2. **框架**：把内容变成网页的程序。用 Next.js 写的，你平时不用管它

当你运行 `npm run build`（构建）时，框架会读取你的 `.md` 文件，把里面的 Markdown 文字转换成 HTML 网页，输出到 `out/` 文件夹。这个文件夹里的东西上传到腾讯云 COS，互联网上的人就能访问了。

**你只需要关心内容，不需要懂代码。** 本文档就是教你如何改内容。

### 核心概念：Markdown 是什么

Markdown 是一种用纯文字写排版的语法。它比 Word 简单——不用点按钮选择"加粗""标题"，而是用符号来表示。

举个例子：

```
你在 .md 文件里写      →    网页上显示为

**这段话是粗体**        →    <strong>这段话是粗体</strong>
# 这是大标题           →    <h1>这是大标题</h1>
```

好处是：文件体积小、任何电脑都能打开、可以用 git 追踪每次修改。这也是为什么大多数技术文档和博客都用 Markdown 写。

## 快速上手：改内容三步

### 第一步：找到要改的文件

打开 VS Code，在左侧文件列表找到 `site/content/pages/` 目录：

```
site/content/pages/
├── about.md        ← 点这个 → 网站上的"关于"页面
├── beliefs.md      ← 点这个 → 网站上的"观念"页面
├── works.md        ← 点这个 → 网站上的"作品"页面
├── favorites.md    ← 点这个 → 网站上的"喜爱"页面
├── changelog.md    ← 点这个 → 网站上的"更新"页面
└── misc.md         ← 点这个 → 网站上的"杂项"页面
```

**对照关系**：文件名 = 网站栏目。`about.md` 就是"关于"页面，`works.md` 就是"作品"页面。

### 第二步：改内容

打开文件，直接用 Markdown 语法写。把你原来写的内容删掉，换成你自己的。

文件第一行 `# 标题` 就是这个页面的标题。后面就是正文。

### 第三步：预览效果

在终端运行：

```bash
npm run dev
```

然后浏览器打开 `http://localhost:3099`。你每次保存 `.md` 文件，浏览器自动刷新，马上能看到效果。

确认没问题后，用 git 保存：

```bash
git add -A
git commit -m "更新了xxx内容"
```

## 各栏目详细说明

### 首页（`src/app/page.tsx`）

首页**不是 `.md` 文件**。它是 React 代码写的，因为首页的布局比较复杂——有 Hero 大标题、有 6 张导航卡片、有动画效果。

但你可以改首页的**文字内容**而不需要懂 React。具体怎么改见后面的"常见操作"章节。

### 关于（`content/pages/about.md`）

写你的个人介绍。建议包含：
- 你是谁、做什么的
- 经历（教育、工作）
- 技能
- 联系方式

### 观念（`content/pages/beliefs.md`）

写你对人生基本问题的看法。目前模板包含：
- 世界观（如何看待世界）
- 人生观（人应该怎样度过一生）
- 价值观（什么是对的、重要的）
- 关于爱情、友情、亲情

你可以保留这些分类，也可以删掉重写。

### 作品（`content/pages/works.md`）

展示你做的东西。模板分为：
- 文稿（文章、笔记、翻译）
- 设计作品
- 其他项目

这个页面特别适合放图片和视频。用法见下文"插入图片"和"插入视频"章节。

### 喜爱（`content/pages/favorites.md`）

分享你喜欢的电影、音乐、书籍、活动。模板用列表格式，你可以改成任何你喜欢的形式。

### 更新日志（`content/pages/changelog.md`）

记录网站的改动。每当你修改了网站（加了新栏目、改了设计、加了新功能），在这里加一条记录。

建议格式：`## 日期 — 标题`，下面用 `-` 列表写具体改动。

### 杂项（`content/pages/misc.md`）

暂时归不了类的内容。如果一个话题积累了足够多的内容，可以升级为独立栏目。

## Markdown 写作语法详解

以下是你在 `.md` 文件中可以使用的所有语法。每个语法都配有"你写什么"和"看到什么"的对照。

### 标题

标题有三级。`#` 的数量决定层级，`#` 越少标题越大。

```markdown
# 一级标题（页面主标题，最大）
## 二级标题（章节标题）
### 三级标题（小节标题）
```

**效果**：

> # 一级标题（页面主标题，最大）
> ## 二级标题（章节标题）
> ### 三级标题（小节标题）

**使用建议**：每个 `.md` 文件的第一行用 `#` 做主标题。正文里用 `##` 分章节。`###` 做小节。

### 段落和换行

```markdown
这是第一个段落。连续的文字会自动连成一段。

这是第二个段落。中间空一行就是新的段落。
（空行是关键，不空行的话这两行会连在一起）
```

**规则**：两个段落之间必须空一行。单按回车不会换行，空一行才会。

### 加粗和斜体

```markdown
**这是加粗文字**，通常用来强调。

*这是斜体文字*，通常用来引用书名、外来语。

***加粗 + 斜体***
```

**效果**：**这是加粗文字**，*这是斜体文字*。

### 链接

```markdown
[显示的文字](链接地址)

例如：
[访问我的 GitHub](https://github.com/your-username)
```

**效果**：[访问我的 GitHub](https://github.com/your-username)

方括号里是读者看到的文字，圆括号里是点击后跳转的网址。

如果是站内链接（跳到你网站的另一个页面），可以写相对路径：

```markdown
[查看作品](/works)
[查看关于](/about)
```

### 无序列表

```markdown
- 第一项
- 第二项
- 第三项
  - 缩进两个空格可以嵌套子项
  - 继续嵌套
```

**效果**：
- 第一项
- 第二项
- 第三项
  - 缩进两个空格可以嵌套子项

用 `-` 或 `*` 开头都可以，加一个空格后写内容。

### 有序列表

```markdown
1. 第一步
2. 第二步
3. 第三步
```

**效果**：
1. 第一步
2. 第二步
3. 第三步

数字不需要连续，Markdown 会自动编号。写 `1. 1. 1.` 也会显示为 1. 2. 3.。

### 引用

```markdown
> 这是一段引用文字。
> 可以多行，每行前面加 > 。
> 
> 空行也要加 > ，否则引用会断开。
```

**效果**：

> 这是一段引用文字。可以多行。适合引用别人的话、或者突出某段文字。

### 分割线

```markdown
上面是前一个话题。

---

下面是新话题。
```

三个或更多 `-`（或 `*`）单独占一行，就会渲染成一条横线。用来在视觉上分隔不同主题。

### 行内代码

```markdown
在段落中使用 `const x = 1` 这样的行内代码标记。
```

**效果**：在段落中使用 `const x = 1` 这样的行内代码标记。

用单个反引号 `` ` `` 包裹。适合在正文中提到变量名、文件名、命令等。

### 代码块

````markdown
```python
def hello():
    print("Hello, World!")
    return True
```
````

三个反引号开始，三个反引号结束。第一个 ``` 后面可以写语言名称（python、javascript、html、css 等），代码会根据语言做语法高亮。

**效果**：
```python
def hello():
    print("Hello, World!")
    return True
```

**不写语言名也可以**：
````markdown
```
任意纯文本
都可以放这里
```
````

### 表格

```markdown
| 名称   | 年份 | 描述           |
|--------|------|---------------|
| 项目A  | 2024 | 这是一个描述    |
| 项目B  | 2023 | 另一个描述     |
```

**效果**：

| 名称   | 年份 | 描述           |
|--------|------|---------------|
| 项目A  | 2024 | 这是一个描述    |
| 项目B  | 2023 | 另一个描述     |

**说明**：
- 第一行是表头
- 第二行是分隔线（`|--------|------|`），必须有
- 第三行开始是数据
- 列数可以增减，竖线对齐不是必须的，但对齐了更好看

### 插入图片

```markdown
![图片描述](图片地址)
```

**分两种方式**：

#### 方式一：本地图片（推荐）

1. 把图片文件放到 `site/public/images/` 目录
2. 在 `.md` 文件中引用：

```markdown
![我的照片](/images/photo.jpg)
![设计稿预览](/images/design-final.png)
```

路径规则：`public/images/xxx.jpg` → 引用 `/images/xxx.jpg`

#### 方式二：远程图片

图片已经在其他网站上，不需要下载：

```markdown
![示例图片](https://example.com/image.jpg)
```

**图片特性**：
- 自动懒加载（滚动到附近才加载，不影响页面速度）
- 自带宽圆角
- 如果写了描述文字，会在图片下方居中显示灰色说明文字
- 支持 jpg、png、webp、svg、gif 等常见格式

### 插入视频

网站支持三种视频方式：

#### 本地视频文件

把视频放到 `site/public/videos/`，然后在 `.md` 中写：

```markdown
!video[演示视频](/videos/demo.mp4)
```

#### Bilibili 视频

打开 Bilibili 视频页面，从网址中复制 BV 号。例如：
- 网址：`https://www.bilibili.com/video/BV1xx411c7mD`
- BV 号：`BV1xx411c7mD`

```markdown
!bilibili[B站视频标题](BV1xx411c7mD)
```

#### YouTube 视频

打开 YouTube 视频页面，从网址中复制视频 ID。例如：
- 网址：`https://www.youtube.com/watch?v=dQw4w9WgXcQ`
- 视频 ID：`dQw4w9WgXcQ`

```markdown
!youtube[YouTube视频标题](dQw4w9WgXcQ)
```

**视频特性**：
- 以 16:9 响应式容器展示（手机和电脑上比例都正确）
- 支持全屏播放
- Bilibili 和 YouTube 用各自的官方播放器
- 标题可选，不写也行：`!bilibili[](BV号)` 或 `!youtube[](视频ID)`

## 常见操作（一步步跟着做）

### 操作 1：改首页的个人介绍

**涉及文件**：`site/src/app/page.tsx`

**步骤**：

1. 打开 `site/src/app/page.tsx`
2. 找到这段代码（在文件中间位置）：

```tsx
<h1 className="...">
  你好，我是{" "}
  <span className="...">
    [你的名字]        ← 把这里改成你的名字
  </span>
</h1>
<p className="...">
  这里是我的数字花园。记录思考，展示作品，整理一切塑造了我的事物。
  ↑ 把这段话改成你自己的介绍
</p>
```

3. 下面还有两个按钮的文字，可以改：

```tsx
<Link href="/about" ...>
  了解更多          ← 改按钮文字
</Link>
<Link href="/works" ...>
  查看作品          ← 改按钮文字
</Link>
```

4. 如果你还想改首页的栏目卡片文字，找到 `sections` 数组（在文件开头）：

```tsx
const sections = [
  {
    href: "/about",
    title: "个人简介",     ← 卡片标题
    desc: "经历、身份、在做的事。",  ← 卡片描述
    // ...
  },
  // ... 后面 5 张卡片类似
];
```

### 操作 2：改网站标题（浏览器标签上显示的文字）

**涉及文件**：`site/src/app/layout.tsx`

打开文件，找到第 9 行附近：

```tsx
export const metadata: Metadata = {
  title: "我的网站",        // ← 改成你的网站名
  description: "个人网站 —— 记录思考、作品和值得留下的东西。",
  // ↑ 改成你的网站描述（搜索引擎会显示这段文字）
};
```

### 操作 3：改导航栏的标签名

**涉及文件**：`site/src/components/Header.tsx`

打开文件，找到 `navItems` 数组：

```tsx
const navItems = [
  { href: "/", label: "首页" },
  { href: "/about", label: "关于" },      // ← 把 "关于" 改成你想要的名字
  { href: "/beliefs", label: "观念" },    // ← 同上
  { href: "/works", label: "作品" },
  { href: "/favorites", label: "喜爱" },
  { href: "/changelog", label: "更新" },
  { href: "/misc", label: "杂项" },
];
```

**只改 `label` 的值**，不要改 `href`。`href` 是网址路径，`label` 是导航栏显示的文字。

### 操作 4：修改配色

**涉及文件**：`site/src/app/globals.css`

打开文件，最顶部就是配色：

```css
:root {
  --accent: #2563eb;        /* 强调色：链接、按钮、高亮 */
  --bg: #fafaf8;            /* 页面背景色 */
  --text: #1a1a1a;          /* 正文文字颜色 */
  --text-muted: #6b6b6b;    /* 辅助文字（灰色） */
  --text-soft: #94948c;     /* 更淡的文字 */
  --border: #e8e8e4;        /* 边框颜色 */
}
```

颜色值用十六进制表示：`#RRGGBB`。
- `#2563eb` = 蓝色
- `#059669` = 绿色
- `#7c3aed` = 紫色
- `#d97706` = 橙色
- `#dc2626` = 红色

可以用 [Coolors](https://coolors.co) 或 [huemint](https://huemint.com) 选配色，选好后替换这里的值即可。

### 操作 5：添加新栏目

如果你想增加一个新栏目（比如"摄影"），需要改 4 个地方：

**步骤 1**：创建内容文件

在 `site/content/pages/` 新建 `photography.md`：

```markdown
# 摄影

这里放你的摄影作品。
```

**步骤 2**：创建页面路由

新建文件夹 `site/src/app/photography/`，在里面创建 `page.tsx`：

```tsx
import { getPageContent } from "@/lib/content";
import { MarkdownRenderer } from "@/components/MarkdownRenderer";
import ScrollReveal from "@/components/ScrollReveal";

export default function PhotographyPage() {
  const { title, content } = getPageContent("photography");

  return (
    <div className="mx-auto max-w-3xl px-4 py-12 sm:py-16">
      <ScrollReveal>
        <p className="text-sm font-medium tracking-wide text-[var(--accent)] uppercase">
          摄影
        </p>
        <h1 className="mt-3 font-serif text-3xl font-bold tracking-tight sm:text-4xl">
          {title}
        </h1>
      </ScrollReveal>
      <ScrollReveal delay={100}>
        <div className="prose mt-10">
          <MarkdownRenderer content={content} />
        </div>
      </ScrollReveal>
    </div>
  );
}
```

把 "摄影" 替换为你的栏目名，把 `getPageContent("photography")` 中的 `"photography"` 改为你的文件名（不含 `.md`）。

**步骤 3**：在导航栏添加入口

打开 `src/components/Header.tsx`，在 `navItems` 数组中添加：

```tsx
{ href: "/photography", label: "摄影" },
```

**步骤 4**：在首页添加卡片

打开 `src/app/page.tsx`，在 `sections` 数组中添加：

```tsx
{
  href: "/photography",
  title: "摄影",
  desc: "用镜头记录下来的瞬间。",
  color: "#d97706",
  icon: (
    <svg width="22" height="22" viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth="1.8">
```

## 常见问题

### 问：为什么我改了 .md 文件，但网站上没变化？

答：检查以下几点：
1. 是否保存了文件（VS Code 标签上是否有圆点）
2. 开发服务器是否在运行（终端里 `npm run dev` 是否还开着）
3. 改的是不是正确的文件（文件名对应哪个页面，见上面的"各栏目详细说明"）
4. 浏览器是否需要刷新（正常会自动刷新，但偶尔需要手动刷新）

### 问：图片不显示怎么办？

答：最常见的原因是路径写错了：
- `public/images/photo.jpg` → 引用时写 `/images/photo.jpg`（不是 `/public/images/photo.jpg`）
- 注意大小写，`Photo.jpg` 和 `photo.jpg` 是不同的
- 图片文件名建议用英文和数字，不要用中文和空格

### 问：我可以改 `.md` 文件的文件名吗？

答：可以，但需要同时修改对应的 `page.tsx` 中的 `getPageContent("xxx")` 调用，把 `"xxx"` 改为新文件名（不含 `.md`）。

### 问：为什么首页不能用 `.md` 写？

答：首页布局比较复杂——有 Hero 区域、有多张卡片、有动画。这些需要 React 代码控制。但首页要改的文字都在 `page.tsx` 中很容易找到的位置。

### 问：`.md` 文件和 `.mdx` 文件有什么区别？

答：`.md` 是纯 Markdown，`.mdx` 可以在 Markdown 中嵌入 React 组件。目前网站用的是 `.md`。以后如果需要更复杂的内容展示，可以切换到 `.mdx`。

### 问：我需要学 React / TypeScript 吗？

答：不需要。90% 的日常操作就是改 `.md` 文件里的文字。只有添加新栏目、改首页结构、改导航栏时才需要改 `.tsx` 代码，但本文档已经把步骤写清楚了，照着做就行。
