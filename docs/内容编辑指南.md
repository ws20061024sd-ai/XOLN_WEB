# 内容编辑指南

> 本文档随网站变化实时更新。最后一次大更新：列表+详情架构上线，frontmatter 系统启用。

## 先理解：这个网站是怎么工作的

你的网站由两个东西组成：

1. **内容**：你写的文字、放的图片。存在 `content/` 目录下的 `.md` 文件里
2. **框架**：把内容变成网页的程序。Next.js，你平时不用管它

构建时，框架读取 `content/` 下的所有 `.md` 文件，根据 frontmatter（文件顶部的元数据）自动生成列表页和详情页。

**你只需要写 .md 文件，不需要懂代码。**

│
├── beliefs/                  ← "观念"栏目
│   ├── worldview.md          ← 每篇一个 .md 文件
│   ├── life-philosophy.md
│   ├── values.md
│   ├── love.md
│   ├── family.md
│   └── friendship.md
│
├── works/                    ← "作品"栏目
│   ├── article-one.md
│   ├── article-two.md
│   └── design-project.md
│
├── favorites/                ← "喜爱"栏目
│   ├── movies.md
│   ├── music.md
│   ├── books.md
│   └── activities.md
│
├── changelog/                ← "更新日志"栏目
│   └── 2025-01-15.md
│

**核心规则**：
- 一个栏目 = `content/` 下的一个文件夹
- 一篇文章 = 文件夹里的一个 `.md` 文件
- 构建时自动生成：列表页（显示所有文章的标题和简介）+ 详情页（点进去看全文）

## 一篇文章的结构（frontmatter + 正文）

每篇 `.md` 文件顶部必须有一段 frontmatter，用 `---` 包裹：

```yaml
---
title: "文章标题"          ← 必填，列表页和详情页的标题
date: "2025-01-15"         ← 必填，日期格式 YYYY-MM-DD
description: "文章简介"    ← 必填，列表页显示的摘要文字
tags: ["标签1", "标签2"]    ← 可选，列表页显示为小标签
order: 1                   ← 可选，数字越小越靠前（详见"排序"章节）
---

正文写在这里。从 frontmatter 下面开始，用 Markdown 语法。
```

### 各字段详解

| 字段 | 是否必填 | 作用 | 示例 |
|------|---------|------|------|
| `title` | 必填 | 列表页标题 + 详情页大标题 | `"世界观"` |
| `date` | 必填 | 列表页显示日期，决定默认排序 | `"2025-01-15"` |
| `description` | 必填 | 列表页摘要，搜索关键词来源 | `"我如何看待这个世界。"` |
| `tags` | 可选 | 列表页显示彩色小标签，搜索可命中 | `["思考", "世界观"]` |
| `order` | 可选 | 自定义排序，数字越小越靠前 | `1` |

## 常见操作

### 操作 1：修改已有文章

1. 打开 `content/栏目名/文件名.md`
2. 修改 frontmatter 或正文
3. 保存 → 浏览器自动刷新看效果

### 操作 2：新增一篇文章

以"作品"栏目为例：

**第 1 步**：在 `content/works/` 下新建一个 `.md` 文件，比如 `my-new-work.md`

**第 2 步**：写 frontmatter + 正文：

```yaml
---
title: "我的新作品"
date: "2025-02-01"
description: "这是一个新作品的简介。"
tags: ["设计", "2025"]
---

# 我的新作品

正文内容在这里写。

## 背景

为什么做这个作品。

## 过程

怎么做的。

![预览图](/images/preview.jpg)
```

**第 3 步**：保存。列表页自动出现，详情页自动生成。

> **文件名建议**：用英文小写 + 连字符，如 `my-new-work.md`。不要用中文和空格。文件名会自动变成 URL：`/works/my-new-work`。

### 操作 3：删除一篇文章

直接删除 `.md` 文件即可。下次构建时列表页和详情页自动消失。

### 操作 4：控制列表排序

**默认排序**：按 `date` 降序（日期新的在前）。

**自定义排序**：给栏目内所有文章加上 `order` 字段：

```yaml
---
title: "世界观"
order: 1      ← 排第一
---

---
title: "人生观"
order: 2      ← 排第二
---
```

规则：
- 栏目内**全部文章都有 order** → 按 order 升序排（1, 2, 3...）
- **任何一篇没有 order** → 回退到按日期降序排

目前只有"观念"栏目用了 order 排序，顺序为：世界观(1) → 人生观(2) → 价值观(3) → 爱情(4) → 亲情(5) → 友情(6)。

### 操作 5：改首页自我介绍

编辑 `site/src/app/page.tsx`，找到：

```tsx
<span className="...">[你的名字]</span>   // ← 名字

<p className="...">
  这里是我的数字花园。记录思考，展示作品...
  // ↑ 介绍文字
</p>
```

### 操作 6：改网站标题

编辑 `site/src/app/layout.tsx`：

```tsx
export const metadata: Metadata = {
  title: "我的网站",   // ← 浏览器标签上显示
};
```

### 操作 7：改导航栏标签

编辑 `site/src/components/Header.tsx`，修改 `navItems` 数组中的 `label`：

```tsx
const navItems = [
  { href: "/", label: "首页" },
  { href: "/about", label: "关于" },      // ← 改这里
  { href: "/beliefs", label: "观念" },
  // ...
];
```

**不要改 `href`**，那是网址路径。

### 操作 8：修改配色

编辑 `site/src/app/globals.css`：

```css
:root {
  --accent: #2563eb;        /* 强调色 */
  --bg: #fafaf8;            /* 背景色 */
  --text: #1a1a1a;          /* 文字色 */
}
```

## Markdown 写作语法

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

### 段落

段落之间空一行。单按回车不换行。

### 文字样式

```markdown
**加粗**
*斜体*
`行内代码`
```

### 链接

```markdown
[文字](网址)
[站内链接](/works)
```

### 列表

```markdown
- 无序列表
- 无序列表

1. 有序列表
2. 有序列表
```

### 引用

```markdown
> 引用文字
```

### 分割线

```markdown
---
```

### 代码块

````markdown
```python
print("hello")
```
````

### 表格

```markdown
| 列A | 列B |
|-----|-----|
| 数据 | 数据 |
```

### 图片

```markdown
![描述](/images/xxx.jpg)             ← 本地（放 public/images/）
![描述](https://example.com/a.jpg)   ← 远程
```

### 视频

```markdown
!video[标题](/videos/demo.mp4)        ← 本地视频（public/videos/）
!bilibili[标题](BV号)                 ← B 站视频
!youtube[标题](视频ID)                ← YouTube 视频
```

详细的图片和视频用法见 [图片与视频使用指南](./图片与视频使用指南.md)。

### 完整的语法测试页

打开 `/misc/md-test`，对照 `content/misc/md-test.md` 源码看所有语法的渲染效果。

## 搜索

搜索页面 `/search` 可以搜索全站所有文章。搜索范围：标题、简介、标签。

- 支持模糊匹配（输入部分文字即可）
- 结果按匹配度排列
- 点击结果直接跳转到详情页

## 常见问题

### 新增文章后列表页没出现？

检查：
1. frontmatter 是否用 `---` 正确包裹
2. `title` 和 `date` 是否填写
3. 文件后缀是否为 `.md`
4. 是否放对了栏目文件夹

### 列表顺序不对？

检查栏目内是否所有文章都有 `order` 字段。混用 order 和无 order 会回退到日期排序。

### 图片不显示？

- 本地图片：检查是否放在 `public/images/`，引用路径是否以 `/images/` 开头
- 远程图片：检查 URL 是否完整可访问

### 改首页为什么不能用 .md？

首页布局复杂——Hero 区域、多张卡片、动画——需要 React 代码控制。但首页要改的文字都在 `page.tsx` 中容易找到的位置。
