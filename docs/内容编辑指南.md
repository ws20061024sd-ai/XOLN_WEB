# 内容编辑指南

> 本文档随网站变化实时更新。当前状态：作品栏目已支持多级文件夹分类，共创栏目已上线。

## 先理解：这个网站是怎么工作的

你的网站由两个东西组成：

1. **内容**：你写的文字、放的图片。存在 `content/` 目录下的 `.md` 文件里
2. **框架**：把内容变成网页的程序。Next.js，你平时不用管它

构建时，框架读取 `content/` 下的所有 `.md` 文件，根据 frontmatter（文件顶部的元数据）自动生成列表页和详情页。

**你只需要写 .md 文件，不需要懂代码。**

## 文件结构一览

```
site/content/
├── about.md                  ← "关于"页面（单页）
│
├── beliefs/                  ← "观念"栏目
│   ├── worldview.md
│   ├── life-philosophy.md
│   ├── values.md
│   ├── love.md
│   ├── family.md
│   └── friendship.md
│
├── works/                    ← "作品"栏目（支持多级文件夹分类）
│   ├── 设计/                 ← 系列 = 子目录
│   │   ├── logo-design.md
│   │   └── poster.md
│   ├── 文稿/                 ← 另一个系列
│   │   ├── essay-one.md
│   │   └── essay-two.md
│   └── 独立作品.md           ← 不属于任何系列的放根目录
│
├── favorites/                ← "喜爱"栏目
│   ├── movies.md
│   ├── music.md
│   ├── books.md
│   └── activities.md
│
├── changelog/                ← "更新日志"栏目
│   ├── 01-init.md
│   ├── 02-visual.md
│   └── ...
│
├── misc/                     ← "杂项"栏目
│   └── md-test.md
│
└── countdowns.json           ← 倒计时数据文件
```

**核心规则**：
- 一个栏目 = `content/` 下的一个文件夹
- 一篇文章 = 文件夹里的一个 `.md` 文件
- 构建时自动生成：列表页（显示所有文章的标题和简介）+ 详情页（点进去看全文）
- **作品栏目特殊规则**：子目录 = 系列分类，可无限嵌套

## 一篇文章的结构（frontmatter + 正文）

每篇 `.md` 文件顶部必须有一段 frontmatter，用 `---` 包裹：

```yaml
---
title: "文章标题"          ← 必填，列表页和详情页的标题
date: "2025-01-15"         ← 必填，日期格式 YYYY-MM-DD
description: "文章简介"    ← 必填，列表页显示的摘要文字
tags: ["标签1", "标签2"]    ← 可选，列表页显示为小标签
order: 1                   ← 可选，数字越小越靠前（详见"排序"章节）
---

正文写在这里。从 frontmatter 下面开始，用 Markdown 语法。
```

### 各字段详解

| 字段 | 是否必填 | 作用 | 示例 |
|------|---------|------|------|
| `title` | 必填 | 列表页标题 + 详情页大标题 | `"世界观"` |
| `date` | 必填 | 列表页显示日期，决定默认排序 | `"2025-01-15"` |
| `description` | 必填 | 列表页摘要，搜索关键词来源 | `"我如何看待这个世界。"` |
| `tags` | 可选 | 列表页显示彩色小标签，搜索可命中 | `["思考", "世界观"]` |
| `order` | 可选 | 自定义排序，数字越小越靠前 | `1` |

## 常见操作

### 操作 1：修改已有文章

1. 打开 `content/栏目名/文件名.md`
2. 修改 frontmatter 或正文
3. 保存 → 浏览器自动刷新看效果

### 操作 2：新增一篇文章（普通栏目）

以"观念"栏目为例：

1. 在 `content/beliefs/` 下新建 `.md` 文件
2. 写 frontmatter + 正文
3. 保存 → 自动出现在列表页和详情页

> **文件名建议**：用英文小写 + 连字符，如 `my-essay.md`。文件名会自动变成 URL。

### 操作 3：作品栏目——创建新系列

**第 1 步**：在 `content/works/` 下新建一个文件夹，文件夹名就是系列名

```
content/works/摄影作品/   ← 新建这个文件夹
```

**第 2 步**：在文件夹里放 `.md` 文件

```
content/works/摄影作品/
├── 街头随拍.md
└── 自然风光.md
```

**第 3 步**：保存。`/works` 页面会自动出现"摄影作品"文件夹卡片，点进去看到两篇文章。

- 系列名用中文即可，会出现在 URL（`/works/摄影作品/街头随拍`）和面包屑导航里
- 系列文件夹可以嵌套（`content/works/摄影/黑白/xxx.md` → `/works/摄影/黑白/xxx`）
- 不想归类的文章直接放 `content/works/` 根目录即可

### 操作 4：删除一篇文章

直接删除 `.md` 文件。删除整个系列文件夹则整个系列消失。

### 操作 5：控制列表排序

**默认排序**：按 `date` 降序（日期新的在前）。

**自定义排序**：给栏目内所有文章加上 `order` 字段：

```yaml
---
title: "世界观"
order: 1      ← 排第一
---

---
title: "人生观"
order: 2      ← 排第二
---
```

规则：
- 栏目内**全部文章都有 order** → 按 order 升序排（1, 2, 3...）
- **任何一篇没有 order** → 回退到按日期降序排
- 作品栏目中：系列文件夹排在文件前面，文件夹之间按名称排序

### 操作 6：改首页自我介绍

编辑 `site/src/app/page.tsx`，找到：

```tsx
<span className="...">[你的名字]</span>   // ← 名字

<p className="...">
  这里是我的数字花园。记录思考，展示作品...
  // ↑ 介绍文字
</p>
```

### 操作 7：改网站标题

编辑 `site/src/app/layout.tsx`：

```tsx
export const metadata: Metadata = {
  title: "我的网站",   // ← 浏览器标签上显示
};
```

### 操作 8：改导航栏标签

编辑 `site/src/components/Header.tsx`，修改 `navItems` 数组中的 `label`：

```tsx
const navItems = [
  { href: "/", label: "首页" },
  { href: "/about", label: "关于" },
  { href: "/beliefs", label: "观念" },
  // ...
];
```

**不要改 `href`**，那是网址路径。

### 操作 9：修改倒计时

编辑 `site/content/countdowns.json`：

```json
[
  { "title": "个人网站上线", "date": "2025-01-15", "type": "since" },
  { "title": "下一个生日", "date": "2025-08-20", "type": "until" }
]
```

- `"type": "since"` → 显示"距今 X 天"
- `"type": "until"` → 显示"还有 X 天"（倒计时）

### 操作 10：修改配色

编辑 `site/src/app/globals.css`：

```css
:root {
  --accent: #2563eb;        /* 强调色 */
  --bg: #fafaf8;            /* 背景色 */
  --text: #1a1a1a;          /* 文字色 */
}
```

## Markdown 写作语法

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

### 段落

段落之间空一行。单按回车不换行。

### 文字样式

```markdown
**加粗**
*斜体*
`行内代码`
```

### 链接

```markdown
[文字](网址)
[站内链接](/works)
```

### 列表

```markdown
- 无序列表
- 无序列表

1. 有序列表
2. 有序列表
```

### 引用

```markdown
> 引用文字
```

### 分割线

```markdown
---
```

### 代码块

````markdown
```python
print("hello")
```
````

### 表格

```markdown
| 列A | 列B |
|-----|-----|
| 数据 | 数据 |
```

### 图片

```markdown
![描述](/images/xxx.jpg)             ← 本地（放 public/images/）
![描述](https://example.com/a.jpg)   ← 远程
```

### 视频

```markdown
!video[标题](/videos/demo.mp4)        ← 本地视频（public/videos/）
!bilibili[标题](BV号)                 ← B 站视频
!youtube[标题](视频ID)                ← YouTube 视频
```

详细的图片和视频用法见 [图片与视频使用指南](./图片与视频使用指南.md)。

### 完整的语法测试页

打开 `/misc/md-test`，对照 `content/misc/md-test.md` 源码看所有语法的渲染效果。

## 搜索

搜索页面 `/search` 可以搜索全站所有文章。搜索范围：标题、简介、标签。

- 支持模糊匹配（输入部分文字即可）
- 结果按匹配度排列
- 点击结果直接跳转到详情页

---

## 评论

所有详情页底部自动有评论区（包括共创文章），不需要任何配置。新建 `.md` 文件后，详情页下面自动带上评论区。

- 读者填写昵称和内容，点提交即可
- 提交后立即可见（无审核）
- 评论按时间倒序排列

管理面板可查看/删除评论。

---

## 共创栏目

`/community` 是读者投稿栏目，不同于你自己写的内容。

- 读者在页面上填写标题、名字、正文，用 Markdown 写作
- 表单旁有 Markdown 语法速查，支持实时预览
- 提交后默认待审核，你在管理面板"共创"Tab 审核通过后才公开
- 每篇共创文章底部自带评论区
- 文章存储在 API 数据库中，不经过本地 `.md` 文件

## 留言板

`/guestbook` 页面，读者填写昵称和留言内容即可。所有人的留言公开可见。

## 暗色模式

导航栏右侧有月亮/太阳图标，点击切换亮色/暗色。默认亮色，手动切换到暗色后浏览器记住选择。

## 管理面板

访问 `/admin` 或点页脚 © 符号进入。输入密钥（默认 `admin123`）登录后可以：

- **评论管理**：查看/隐藏/删除
- **留言板管理**：查看/删除
- **共创审核**：预览正文、审核通过、隐藏、删除
- **消息查看**：联系表单提交的消息
- **统计查看**：总访问量、今日访问量、页面排名

## 常见问题

### 新增文章后列表页没出现？

检查：
1. frontmatter 是否用 `---` 正确包裹
2. `title` 和 `date` 是否填写
3. 文件后缀是否为 `.md`
4. 是否放对了栏目文件夹

### 作品栏目文章不显示？

检查：
1. 文件是否放在 `content/works/` 或其子目录下
2. 子目录名是否以下划线开头（`_` 开头的目录会被忽略）

### 列表顺序不对？

检查栏目内是否所有文章都有 `order` 字段。混用 order 和无 order 会回退到日期排序。

### 图片不显示？

- 本地图片：检查是否放在 `public/images/`，引用路径是否以 `/images/` 开头
- 远程图片：检查 URL 是否完整可访问
