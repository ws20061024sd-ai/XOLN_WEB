# 后端架构方案

> Phase 2 启用。目前前端纯静态已就绪，本文档规划后续动态功能的架构和实现。

## 整体架构

```
用户浏览器
    │
    ├── yourdomain.com        → 腾讯云 COS（静态前端，已就绪）
    │   获取 HTML/CSS/JS
    │
    └── api.yourdomain.com    → 腾讯云 Lighthouse（动态 API，Phase 2）
        评论 / 统计 / 表单 / ...
```

前端和后端是两个独立的东西，互不绑定：
- 前端在 COS 上，纯静态，永远在线
- 后端在 Lighthouse 上，跑 API 服务
- 前端通过 `fetch` 调用后端，后端挂了不影响页面展示

## Lighthouse 是什么

腾讯云轻量应用服务器。本质上是一台 24 小时开机的云电脑，装了 Linux。

| 对比 | COS（你已在用） | Lighthouse（Phase 2 购买） |
|------|---------------|--------------------------|
| 本质 | 网盘，存文件 | 虚拟机，跑程序 |
| 能干嘛 | 存放 HTML/CSS/JS，浏览器直接访问 | 运行 Node.js、数据库、Nginx |
| 有系统吗 | 无 | Linux |
| 计费 | 按存储和流量，几乎免费 | 按月付固定费用 |
| 月费 | ≈ 0 元 | ≈ 40 元起 |

**推荐配置**：入门型 2核1G，40GB SSD，约 40 元/月。个人博客的 API 完全够用。

## 技术栈

| 组件 | 选择 | 理由 |
|------|------|------|
| API 框架 | Hono | 12KB，路由即类型，内置 CORS/JWT |
| 数据库 | SQLite（better-sqlite3） | 零安装，一个文件就是数据库，备份即复制 |
| 部署 | Docker + Nginx | 一键启动，自动重启，反代 HTTPS |
| 认证 | 管理员密钥（简单场景） | 管理接口用密钥校验，不需要登录系统 |
| 评论认证 | GitHub OAuth（可选） | 读者用 GitHub 账号登录后评论，免去注册 |

### 为什么 SQLite 而不是 MySQL

- SQLite 不需要安装数据库服务，一个 `blog.db` 文件就是整个数据库
- 个人博客体量下性能完全过剩（每秒能处理几万次查询）
- 备份 = 复制文件。迁移 = 复制文件
- Lighthouse 上不需要额外内存跑数据库进程

如果未来数据量真的上去了（几万条评论），可以平滑切换到 Turso（云 SQLite）或 MySQL。

## API 设计

### 评论系统

```
GET    /api/comments/:slug          获取某篇文章的评论列表
POST   /api/comments                提交新评论（需校验）
PATCH  /api/comments/:id/approve    审核通过（管理接口，需密钥）
DELETE /api/comments/:id            删除评论（管理接口，需密钥）
```

**数据库表**：

```sql
CREATE TABLE comments (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  slug        TEXT NOT NULL,          -- 文章标识（如 "worldview"）
  section     TEXT NOT NULL,          -- 栏目（如 "beliefs"）
  author      TEXT NOT NULL,          -- 评论者昵称
  email       TEXT,                   -- 可选，用于通知
  content     TEXT NOT NULL,          -- 评论内容
  approved    INTEGER DEFAULT 0,      -- 0=待审核, 1=已通过
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**反垃圾策略**：
- 提交时校验字段长度（昵称 ≤50字，内容 ≤5000字）
- 同一 IP 每分钟最多发 2 条（Hono 内置限流中间件）
- 所有评论默认 `approved=0`（待审核），你在管理端手动通过
- 前端只展示 `approved=1` 的评论

### 阅读统计

```
POST /api/stats/pageview     记录一次页面访问（前端每个页面加载时调用）
GET  /api/stats/top          获取访问量最高的文章 Top 10
GET  /api/stats/summary      获取总 PV/UV 概览
```

```sql
CREATE TABLE pageviews (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  path      TEXT NOT NULL,       -- 页面路径
  ip        TEXT,                -- 去重用
  user_agent TEXT,               -- 浏览器信息
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

前端每个页面加载时发一个 POST 请求。不需要用户交互，无感知。

### 联系表单

```
POST /api/contact   提交联系表单
```

```sql
CREATE TABLE messages (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  name      TEXT NOT NULL,
  email     TEXT NOT NULL,
  content   TEXT NOT NULL,
  read      INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

提交后存储到数据库，同时发一封邮件通知你（用 Nodemailer + QQ 邮箱/Gmail SMTP）。

## 前端如何调用后端

前端通过一个统一的 API 工具函数调用后端。Phase 1 时 API 地址为空，所有动态功能静默降级：

```ts
// site/src/lib/api.ts
const API_BASE = process.env.NEXT_PUBLIC_API_URL || "";

// 获取评论
export async function fetchComments(slug: string) {
  if (!API_BASE) return [];  // Phase 1：还没部署后端，返回空
  const res = await fetch(`${API_BASE}/api/comments/${slug}`);
  return res.json();
}

// 提交评论
export async function postComment(data: { slug: string; author: string; content: string }) {
  if (!API_BASE) return { ok: false, error: "评论功能暂未开放" };
  const res = await fetch(`${API_BASE}/api/comments`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(data),
  });
  return res.json();
}

// 记录页面访问
export async function trackPageview(path: string) {
  if (!API_BASE) return;
  fetch(`${API_BASE}/api/stats/pageview`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ path }),
  }).catch(() => {}); // 静默失败，不影响浏览
}

// 提交联系表单
export async function submitContact(data: { name: string; email: string; content: string }) {
  if (!API_BASE) return { ok: false, error: "联系功能暂未开放" };
  const res = await fetch(`${API_BASE}/api/contact`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(data),
  });
  return res.json();
}
```

**切换流程**：Phase 2 时在环境变量设置 `NEXT_PUBLIC_API_URL=https://api.yourdomain.com`，重新构建上传 COS，所有动态功能自动激活。

## 部署到 Lighthouse

### 目录结构

```
/srv/blog-api/           # Lighthouse 上的项目目录
├── docker-compose.yml
├── Dockerfile
├── data/                # SQLite 数据库文件挂载到宿主机
├── package.json
├── tsconfig.json
└── src/
    ├── index.ts         # Hono 入口
    ├── routes/
    │   ├── comments.ts
    │   ├── stats.ts
    │   └── contact.ts
    └── db/
        └── schema.ts    # 建表语句
```

### Dockerfile

```dockerfile
FROM node:22-alpine
WORKDIR /app
COPY package*.json .
RUN npm ci
COPY . .
RUN npm run build
EXPOSE 3001
CMD ["node", "dist/index.js"]
```

### docker-compose.yml

```yaml
services:
  api:
    build: .
    ports:
      - "3001:3001"
    volumes:
      - ./data:/app/data     # SQLite 文件持久化
    restart: always
```

### Nginx 反代

```nginx
server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate     /etc/ssl/api.yourdomain.pem;
    ssl_certificate_key /etc/ssl/api.yourdomain.key;

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

SSL 证书用腾讯云免费的一年期证书，或者 Let's Encrypt 自动续期。

### 部署命令

```bash
# 在本地写好代码后
git push

# SSH 进 Lighthouse
ssh root@你的Lighthouse公网IP

# 拉取最新代码
cd /srv/blog-api
git pull

docker compose up -d --build

# 查看状态
docker compose ps
docker compose logs -f
```

| 步骤 | 操作 |
|------|------|
| 1 | 购买 Lighthouse，选入门型，装好 Docker |

## 安全注意事项

- **CORS**：API 只允许 `yourdomain.com` 的请求，防止其他网站恶意调用
- **限流**：每个 IP 每分钟最多发 2 条评论，防刷
- **输入校验**：所有用户输入做长度限制和 XSS 过滤
- **SQL 注入防护**：全部使用参数化查询，不拼接字符串
- **管理接口加密钥**：审核、删除等操作用环境变量校验密钥，不暴露给前端
- **HTTPS**：Nginx 配置 SSL，API 全程加密传输
