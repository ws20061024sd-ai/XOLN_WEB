## 写作流程：怎么改文章最省事

```
写文章 → 本地 .md 文件 → git add → git commit → git push
```

这是最朴素的流程，优势：

- 本地写，不依赖网络，不依赖任何后端
- VS Code 装 MDX 插件后支持实时预览、语法高亮、语法检查
- git 本身就是版本管理，误删可以回退
- 所有文章在本地文件夹里，备份/迁移就是复制一个目录

**便捷性提升**：加一个 npm script 省去手敲命令：

```json
// package.json
{
  "scripts": {
    "new": "node scripts/new-post.mjs",
    "dev": "next dev",
    "build": "next build",
    "deploy": "next build && node scripts/upload.mjs"
  }
}
```

```js
// scripts/new-post.mjs —— 交互式创建文章
import readline from 'readline'
import fs from 'fs'

const rl = readline.createInterface({ input: process.stdin, output: process.stdout })

const ask = (q) => new Promise(resolve => rl.question(q, resolve))

const title = await ask('文章标题: ')
const slug = await ask('slug (留空自动生成): ') || title.toLowerCase().replace(/\s+/g, '-')
const tags = (await ask('标签 (逗号分隔): ')).split(',').map(s => s.trim())
const desc = await ask('摘要: ')

const date = new Date().toISOString().split('T')[0]
const dir = `content/${date.slice(0, 4)}`
fs.mkdirSync(dir, { recursive: true })

const content = `---
title: "${title}"
date: "${date}"
description: "${desc}"
tags: [${tags.map(t => `"${t}"`).join(', ')}]
draft: false
---

开始写...
`

fs.writeFileSync(`${dir}/${slug}.mdx`, content)
console.log(`已创建: ${dir}/${slug}.mdx`)
rl.close()
```

用法：`npm run new` → 回答几个问题 → 生成带 frontmatter 的空白文章 → 开写。

### 方案 B：浏览器直接写（GitHub 在线编辑）

如果临时改一个错别字，不想开 VS Code：

```
浏览器 → GitHub 仓库 → 按 . 键 → VS Code Web → 编辑 → 提交
```

按 `.` 键是 GitHub 隐藏功能，直接在浏览器打开 VS Code 编辑界面。改完点提交，Actions 自动部署。

不需要安装任何东西。

### 方案 C：CMS 管理后台（Phase 2）

如果以后文章多了（50+），本地文件管理开始吃力，可以装 Decap CMS：

```
┌─────────────────────────┐
│ Decap CMS 管理后台      │  ← 浏览器里写文章，富文本/Markdown 编辑
│ yourdomain.com/admin    │
└───────┬─────────────────┘
        │ OAuth → GitHub API
        ▼
┌─────────────────────────┐
│ GitHub 仓库             │  ← 实际存储还是 .mdx 文件
└───────┬─────────────────┘
        │ Actions
        ▼
┌─────────────────────────┐
│ COS 部署                │
└─────────────────────────┘
```

Decap CMS 本质就是一个 SPA 页面，放在 `/admin` 路由下。它用 GitHub API 直接往仓库提交文件。不需要后端，不需要数据库。配置一个 `config.yml` 就行。

**但不建议现在搞。** 写文章这个习惯本身要培养，CMS 是有了习惯之后提速用的。10 篇文章以内，VS Code 比任何 CMS 都快。

### 个人偏好

我的建议：**方案 A 起步，偶尔方案 B，文章数超 30 再考虑方案 C。**

理由是：本地写有最好的编辑体验（VS Code + 插件），git 给你免费的历史版本，而且你不需要在写文章这件事上依赖额外的在线服务。博客可以下线，写作环境不行。

## 当前栏目架构（已实现）

### 路由表

```
yourdomain.com
├── /                    首页
├── /about               关于（单页）
├── /beliefs             观念列表
├── /beliefs/[slug]      观念详情（世界观/人生观/价值观/爱情/亲情/友情）
├── /works               作品列表
├── /works/[slug]        作品详情
├── /favorites           喜爱列表
├── /favorites/[slug]    喜爱详情（电影/音乐/书籍/活动）
├── /changelog           更新列表
├── /changelog/[slug]    更新详情
├── /misc                杂项列表
├── /misc/[slug]         杂项详情（含 MD 语法测试）
├── /countdown           倒计时（实时计时）
├── /guestbook           留言板
├── /search              全站搜索
└── /admin               管理面板（页脚 © 入口）
```
├── /links/             友情链接
├── /about/             关于我
├── /rss.xml            RSS 订阅
└── /search/            搜索页
```

对应的文件结构：

```
src/app/
├── layout.tsx              ← 全局布局（Header + Footer）
├── page.tsx                ← / 首页
├── posts/
│   └── [slug]/
│       └── page.tsx        ← /posts/hello-world
├── tags/
│   └── [tag]/
│       └── page.tsx        ← /tags/nextjs
├── projects/
│   └── page.tsx            ← /projects/ 项目展示
├── links/
│   └── page.tsx            ← /links/ 友链
├── about/
│   └── page.tsx            ← /about/ 关于我
└── search/
    └── page.tsx            ← /search 搜索页
```

### 各栏目的内容来源

| 栏目 | 内容怎么来 | 要不要后端 |
|------|-----------|-----------|
| 首页 | 构建时从 `content/` 读取，静态生成 | 不要 |
| 文章详情 | 构建时从 `.mdx` 编译为 HTML | 不要 |
| 标签页 | 构建时从 frontmatter 聚合 | 不要 |
| 项目展示 | 静态数据（写死在 `page.tsx` 或用 `.mdx`） | 不要 |
| 友链 | 静态数据 | 不要 |
| 关于我 | 静态页面或 `.mdx` | 不要 |
| RSS | 构建时生成 XML | 不要 |
| 搜索 | 客户端 Fuse.js | 不要 |

**全静态，全不要后端。** 这些栏目在构建时就全部生成好了，就是一堆 HTML 文件。

### 首页可以做分区

不只是一个文章列表，可以分几个区域：

```
┌─────────────────────────────────────┐
│  Hero 区域                          │
│  "Hi，我是xxx，这里写博客。"          │
│                                     │
│  ── 最新文章 ──                      │
│  卡片1   卡片2   卡片3               │
│                                     │
│  ── 精选项目 ──                      │
│  项目A   项目B   项目C               │
│                                     │
│  更多文章 →                          │
│  文章1 · 日期                       │
│  文章2 · 日期                       │
│  文章3 · 日期                       │
│  < 1 2 3 > 分页                     │
└─────────────────────────────────────┘
```

实现：首页的 `page.tsx` 里同时渲染文章列表和项目列表，数据都来自构建时。

```tsx
// app/page.tsx
import { getAllPosts } from '@/lib/posts'
import { getProjects } from '@/lib/projects'  // 读 projects/ 目录或写死的数组

export default function Home() {
  const posts = getAllPosts().filter(p => p.featured).slice(0, 3)  // 精选前3
  const allPosts = getAllPosts().slice(3)  // 其余
  const projects = getProjects().slice(0, 3)

  return (
    <>
      {/* Hero */}
      <section>...</section>

      {/* 精选项目 */}
      <section>
        <h2>项目</h2>
        <div className="grid grid-cols-3">
          {projects.map(p => <ProjectCard key={p.slug} {...p} />)}
        </div>
      </section>

      {/* 最新文章 */}
      <section>
        <h2>最新文章</h2>
        {posts.map(p => <PostCard key={p.slug} {...p} />)}
        <Link href="/posts">全部文章 →</Link>
      </section>
    </>
  )
}
```

### 关于我和项目页也可以写 MDX

不只是文章可以用 MDX。关于我页、项目页也可以用 `.mdx` 写，存储位置区分开：

```
content/
├── posts/          ← 文章
│   ├── 2024/
│   └── 2025/
├── projects/       ← 项目介绍（每篇 .mdx 有 frontmatter 写链接/截图）
│   ├── blog.mdx
│   └── another-project.mdx
└── pages/          ← 独立页面（关于我、友链声明等）
    ├── about.mdx
    └── links.mdx
```

这样关于我页想改几个字，直接改 `content/pages/about.mdx` 就行，不用翻 React 代码。

```tsx
// app/about/page.tsx
import { getPageBySlug } from '@/lib/posts'

export default function About() {
  const page = getPageBySlug('about')  // 读 content/pages/about.mdx
  return <article className="prose">{page.content}</article>
}
```

## 内容更新路线总结

```
日常改文章（90% 的情况）：
  VS Code 打开 → 改 → git push → 自动部署
  耗时：30秒

临时改错别字：
  浏览器 github.com → 按 . → 改 → 提交
  耗时：1分钟

创建新文章：
  npm run new → 填标题/标签 → 开始写 → git push
  耗时：30秒（不含写作）

某天想加 CMS（可选）：
  装 Decap CMS → 配 config.yml → /admin 可访问
  耗时：1小时
```

**最便利的关键不是工具，是 git push 自动部署**。只要你配好了 GitHub Actions，推到 main 分支就自动上传 COS。改一个字也是一个 git push，没有额外步骤。
