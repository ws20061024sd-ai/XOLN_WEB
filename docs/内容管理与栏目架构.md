# 内容管理与栏目架构

> 当前状态：前后端完整架构（2026-06-24）。242 个静态页面，18 个 API 接口，共创栏目已上线，作品支持多级文件夹。
> 详细操作见 [内容编辑指南](./内容编辑指南.md)，后端说明见 [后端架构方案](./后端架构方案.md)。

## 当前栏目架构

### 路由表

```
xolnxoln.cn
├── /                      首页（Hero + 8 个栏目卡片）
├── /about                 关于（单页，含联系表单）
├── /beliefs               观念列表
├── /beliefs/[slug]        观念详情（6 篇：世界观/人生观/价值观/爱情/亲情/友情）
├── /works                 作品列表（系列文件夹 + 独立作品）
├── /works/[...path]       作品分类/详情（支持任意深度文件夹嵌套）
├── /favorites             喜爱列表
├── /favorites/[slug]      喜爱详情（4 篇：电影/音乐/书籍/活动）
├── /changelog             更新列表
├── /changelog/[slug]      更新详情（11 篇）
├── /community             共创（读者投稿列表）
├── /community?id=xxx      共创文章详情（客户端渲染）
├── /guestbook             留言板
├── /misc                  杂项列表
├── /misc/[slug]           杂项详情
├── /countdown             倒计时
├── /search                全站搜索
└── /admin                 管理面板（页脚 © 入口）
```

### 各栏目的内容来源

| 栏目 | 内容来源 | 需要后端 |
|------|---------|:---:|
| 首页 | React 组件（`page.tsx`） | ❌ |
| 关于 | `content/about.md`（单页） | ❌ |
| 观念 | `content/beliefs/*.md`，构建时静态生成 | ❌ |
| 作品 | `content/works/**/*.md`（子目录=系列），构建时静态生成 | ❌ |
| 喜爱 | `content/favorites/*.md`，构建时静态生成 | ❌ |
| 更新 | `content/changelog/*.md`，构建时静态生成 | ❌ |
| 杂项 | `content/misc/*.md`，构建时静态生成 | ❌ |
| 共創 | API 数据库，客户端渲染 | ✅ |
| 留言板 | API 数据库，客户端渲染 | ✅ |
| 评论 | API 数据库，客户端渲染 | ✅ |
| 搜索 | 构建时生成索引，客户端模糊搜索 | ❌ |
| 倒计时 | `content/countdowns.json` | ❌ |
| 管理面板 | API 调用 | ✅ |

### 内容文件结构

```
site/
├── content/                    ← 你写内容的地方
│   ├── about.md                ← 关于页
│   ├── countdowns.json         ← 倒计时数据
│   ├── beliefs/                ← 观念（6 篇）
│   ├── works/                  ← 作品（支持子目录=系列）
│   │   ├── 设计/
│   │   │   └── *.md
│   │   ├── 文稿/
│   │   │   └── *.md
│   │   └── 独立作品.md         ← 不归类的放根目录
│   ├── favorites/              ← 喜爱（4 篇）
│   ├── changelog/              ← 更新日志（9 篇）
│   └── misc/                   ← 杂项
├── src/                        ← 前端代码
│   ├── app/                    ← 页面路由（App Router）
│   ├── components/             ← 组件
│   └── lib/                    ← 工具函数（content.ts/api.ts）
├── server/                     ← 后端 API（独立部署）
├── upload.js                   ← COS 上传脚本
└── .env.local                  ← 环境变量
```

## 排序规则

- **观念栏目**：全部有 `order`，按 order 升序排
- **更新栏目**：全部有 `order`，order 越小越新，排在最前
- **作品栏目**：目录在前（按名称升序），文件在后（有 order 按 order，否则按日期降序）
- 其他栏目：有 order 按 order，否则按日期降序

## 写作流程

```
打开 VS Code → 新建/修改 .md 文件 → 保存
→ 本地 localhost:3099 实时预览
→ 确认无误 → git add/commit/push
→ npm run build → node upload.js → CDN 刷新 → 线上生效
```

这就是当前的完整流程。没有 CMS，没有自动化 CI/CD——每次改动都需要手动构建、上传、刷 CDN。
