# 图片与视频使用指南

> 图片和视频功能已全部实现：本地 `public/images/` + `public/videos/`，远程 URL，Bilibili/Youtube 嵌入。

## 图片

### 方式一：本地图片（最推荐）

**第 1 步**：把图片文件放入 `site/public/images/` 目录。

```
site/public/images/
├── avatar.jpg          ← 你的头像
├── design-preview.png  ← 设计稿预览
└── photo-2024.webp     ← 某张照片
```

**第 2 步**：在 `.md` 文件中引用。

```markdown
![我的头像](/images/avatar.jpg)
![设计稿](/images/design-preview.png)
```

**路径对照表**：

| 文件的实际位置 | 在 .md 里怎么写 |
|--------------|---------------|
| `public/images/photo.jpg` | `/images/photo.jpg` |
| `public/images/2024/summer.jpg` | `/images/2024/summer.jpg` |
| `public/logo.svg` | `/logo.svg` |

> `public/` 文件夹里的东西，网站访问时路径从 `/` 开始。**不是** `/public/images/xxx`，是 `/images/xxx`。

### 方式二：远程图片

图片已经在互联网上的某个网址，直接在 `.md` 里写完整 URL：

```markdown
![一张远程图片](https://example.com/photo.jpg)
```

### 如何把自己的图片变成远程图片

如果你想把图片放在别处（不跟网站文件混在一起），有几种选择：

#### 上传到腾讯云 COS

你已经有一个 COS 桶在跑网站。可以建一个文件夹专门放图片：

1. 登录腾讯云 COS 控制台
2. 进入你的存储桶，新建文件夹 `images/`
3. 上传图片到 `images/` 文件夹
4. 获取图片的公开访问 URL（桶设置需开启公共读）
5. 在 `.md` 中引用：

```markdown
![图片](https://你的桶名.cos.ap-guangzhou.myqcloud.com/images/photo.jpg)
```

#### 上传到 GitHub 仓库

1. 在 GitHub 新建一个仓库（比如 `assets`）
2. 上传图片到仓库
3. 点击图片 → 右键复制 raw 链接
4. 获取的 URL 类似：`https://raw.githubusercontent.com/你的用户名/assets/main/photo.jpg`
5. 在 `.md` 中引用：

```markdown
![图片](https://raw.githubusercontent.com/你的用户名/assets/main/photo.jpg)
```

#### 使用图床服务

免费图床，上传即得 URL。适合临时使用：

- [SM.MS](https://sm.ms) —— 免费，单张 5MB 以内
- [ImgURL](https://imgurl.org) —— 免费注册，国内访问快

### 图片格式建议

| 格式 | 适合场景 | 特点 |
|------|---------|------|
| JPG | 照片、复杂图片 | 体积小，不支持透明 |
| PNG | 截图、带文字的图、需要透明背景 | 无损压缩，体积较大 |
| WebP | 所有场景（推荐） | 体积比 JPG 小 30%，比 PNG 小 50%，所有现代浏览器都支持 |
| SVG | Logo、图标、示意图 | 矢量，任意缩放不模糊 |
| GIF | 动图 | 体积大，尽量少用 |

### 图片注意事项

- **文件名建议用英文和数字**，不要用中文和空格。好：`photo-2024.jpg`，不好：`我的照片2024.jpg`
- **图片压缩后再放进去**。一张照片动辄 5-10MB，压缩到 200KB 网页加载速度差几十倍。推荐工具：[Squoosh](https://squoosh.app)
- **图片懒加载是自动的**。`MarkdownRenderer` 会给所有图片加 `loading="lazy"`，滚动到图片附近才加载
- **描述文字会显示为图片下方的灰色说明**。`![这是描述文字](/images/xxx.jpg)` → 图片下方显示"这是描述文字"

## 视频

### 方式一：本地视频文件

**第 1 步**：把视频文件放入 `site/public/videos/` 目录。

```
site/public/videos/
└── demo.mp4
```

**第 2 步**：在 `.md` 文件中引用。

```markdown
!video[演示视频](/videos/demo.mp4)
```

视频会渲染为一个 16:9 的播放器，带控制条（播放、暂停、进度、全屏）。

> 视频文件通常很大（几十 MB 到几百 MB），不建议放太多本地视频。构建和上传 COS 时会很慢。短小的演示视频可以，长的建议用 B 站或 YouTube。

### 方式二：B 站视频嵌入

B 站视频不需要下载，直接嵌入播放器。

1. 打开 B 站视频页面
2. 从网址中复制 BV 号
   - 网址：`https://www.bilibili.com/video/BV1GJ411x7h7`
   - BV 号就是 URL 中 `/video/` 后面的那串：`BV1GJ411x7h7`

3. 在 `.md` 文件中写：

```markdown
!bilibili[视频标题](BV1GJ411x7h7)
```

### 方式三：YouTube 视频嵌入

1. 打开 YouTube 视频页面
2. 从网址中复制视频 ID
   - 网址：`https://www.youtube.com/watch?v=dQw4w9WgXcQ`
   - 视频 ID 就是 `v=` 后面的那串：`dQw4w9WgXcQ`

3. 在 `.md` 文件中写：

```markdown
!youtube[视频标题](dQw4w9WgXcQ)
```

## 快速参考

```markdown
# 图片
![描述](/images/文件名.jpg)                    ← 本地图片
![描述](https://example.com/photo.jpg)        ← 远程图片

# 视频
!video[标题](/videos/文件名.mp4)               ← 本地视频
!bilibili[标题](BV号)                          ← B 站视频
!youtube[标题](视频ID)                         ← YouTube 视频
```
